# 极简主题与暗黑模式设计

## 背景

当前博客是一个基于 Next.js 的个人写作站点，使用 Tailwind CSS v4，并在 `app/globals.css` 中定义设计 token。本地 UI 基础组件位于 `components/ui`。现有视觉语言偏暖色、纸张感和编辑部风格；本次确认的新方向是更清爽、克制的「极简作品集 / 技术文档」风格，并支持暗黑模式切换。

## 目标

- 将站点视觉调整为已确认的「极简作品集 / 技术文档」风格。
- 首次访问时根据用户系统偏好自动选择浅色或暗色。
- 提供手动主题切换按钮，让读者可以覆盖系统偏好。
- 持久化用户手动选择，后续访问继续使用该选择。
- 保持实现轻量，沿用当前项目中的本地组件和样式组织方式。

## 非目标

- 不引入大型主题框架，也不重构站点内容模型。
- 不改变文章解析、归档、搜索、分页或路由行为。
- 不增加浅色和暗色之外的多套主题色板。

## 推荐方案

采用轻量的自管理主题切换方案，不新增依赖。在根布局中加入一段很小的内联脚本，在 React hydration 之前为 `<html>` 设置初始 `dark` class。初始主题选择优先级如下：

1. 优先使用 `localStorage` 中保存的用户手动选择。
2. 如果没有保存选择，则跟随系统 `prefers-color-scheme: dark`。
3. 如果无法读取偏好，则默认使用浅色模式。

新增一个小型客户端组件用于渲染顶部导航中的主题切换按钮。用户点击按钮时，组件会更新 `<html>` 的 `dark` class，并把选择写入 `localStorage`。

这个方案适合当前博客规模：不增加依赖，行为容易检查，也能保留现有以服务端渲染页面为主的结构。

## 视觉方向

新的浅色主题应使用冷静、克制的色彩：

- 背景：白色到淡蓝灰色，使用轻微渐变替代当前暖色纸张感。
- 前景文字：深 slate 或深蓝黑，保证正文阅读清晰。
- 主色：靛蓝或蓝紫色，用于链接、按钮、焦点环和 hover 状态。
- 卡片：以白色为主，配合浅边框、更轻的阴影和更低的暖色感。
- 次要文字：使用 slate 灰，并保持元信息和描述文字的可读对比度。

暗色主题应像安静的技术资料库，而不是纯黑终端：

- 背景：深蓝黑。
- 卡片：略微抬起的深 slate。
- 前景文字：柔和的近白色。
- 主色：在暗色背景上仍然清晰的柔和蓝紫色。
- 边框：低对比度 slate 线条。

字体可以继续沿用当前系统字体栈。最主要的视觉变化来自色彩、间距细节、更轻的卡片处理，以及更像产品界面的顶部主题控制。

## 组件与布局变化

- 更新 `app/globals.css` 中的设计 token，让现有 UI 基础组件自动继承新的浅色和暗色调色板。
- 增加 `.dark` token 覆盖块，用于暗色主题。
- 更新全局背景，让浅色和暗色模式下都自然。
- 增加 `color-scheme` 处理，使原生控件和浏览器默认渲染匹配当前主题。
- 新增 `components/theme-toggle.tsx`，作为小型客户端组件。
- 在 `app/layout.tsx` 的顶部导航旁放置主题切换按钮。
- 保持首页、文章归档页、文章详情页和 404 页的结构不变；只在新风格需要时做少量 class 调整。

## 可访问性与体验

- 主题切换控件应使用真实的 `<button>`，并提供可访问标签。
- 焦点样式继续使用共享的 `ring` token。
- 初始渲染应尽量避免浅色到暗色的闪烁，因此主题 class 需要在 React hydration 前设置。
- 保存的主题只代表用户明确手动选择；首次访问且没有保存选择时，应跟随系统偏好。

## 测试与验证

- 实现后运行 lint。
- 实现后运行现有测试套件。
- 手动验证浅色模式、暗色模式，以及没有保存偏好时的系统跟随行为。
- 至少检查首页、文章归档页和文章详情页，确认对比度、卡片样式、链接、标签和主题切换状态正常。

## 主题切换呈现

顶部导航使用无依赖的文字切换按钮。按钮展示下一步可执行的动作：当前为浅色时显示「暗色」，当前为暗色时显示「浅色」。这样不需要引入图标库，也能保持控件含义清楚。
